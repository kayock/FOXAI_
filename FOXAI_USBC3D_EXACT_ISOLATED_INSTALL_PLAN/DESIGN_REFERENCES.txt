C3D DESIGN REFERENCES

The C3D pip simulation and future C3E command design follow the official pip
interface documented for:

- pip install --dry-run with --ignore-installed and --report
- pip install --target
- pip install --no-deps and --no-index
- pip install --only-binary=:all:
- requirements files containing exact local wheel paths
- per-requirement --hash values and --require-hashes

C3D additionally constrains pip to exact local file:// wheel references and never
passes a target directory during its dry-run.

Official documentation:
https://pip.pypa.io/en/stable/cli/pip_install/
https://pip.pypa.io/en/stable/reference/requirements-file-format/
https://pip.pypa.io/en/stable/cli/pip_hash/

R1 INSTALLER ENGINE POLICY
The protected portable interpreter remains unchanged and has no pip. C3D R1
therefore permits the already verified same-version host CPython only as the
offline pip process for dry-run planning and the future isolated --target
transaction. The final target is verified using the portable interpreter and
must not import packages from the host.
