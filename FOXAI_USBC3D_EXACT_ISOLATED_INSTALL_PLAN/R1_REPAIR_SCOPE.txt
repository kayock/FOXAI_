FOXAI USB C3D R1 — Installer Engine Fallback Repair

FAILED C3D FINDING
The original C3D correctly stopped because the protected portable Python 3.14.6
has no pip module. All other pre-install planning checks passed.

R1 CHANGE
- Portable pip remains the preferred engine.
- When portable pip is absent, C3D may use only:
    C:\Python314\python.exe
    CPython 3.14.6, AMD64, 64-bit Windows
    pip 26.1.2
- The host executable is hashed and its exact identity/capabilities are recorded.
- Python isolated mode (-I) and pip isolated mode (--isolated) are both required.
- C3D still performs only a dry run and passes no installation target.
- The future C3E command uses --ignore-installed so host-installed packages
  cannot suppress installation into the isolated transaction target.
- C3E must re-verify the installer executable hash and pip version before acting.
- The final package target must be tested with the protected portable interpreter
  and must not import from host Python.

UNCHANGED
- 96-wheel manifest, versions, URLs, sizes, and SHA-256 values
- C3B install order
- C3C staging wheelhouse
- preferred final target
- transaction-by-adjacent-staging-and-rename design
- no-network, no-launcher-change, no-ComfyUI-launch boundaries
- no installation or target creation during C3D
