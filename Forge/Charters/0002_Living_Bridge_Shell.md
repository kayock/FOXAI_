# Charter 0002 - Living Bridge Shell

## Purpose

Establish Kayock Browser as the primary Operator-facing shell.

## Reason

KayocktheOS should feel like an operating environment, not a folder of tools.

## Decision

The browser becomes the Shell. Python remains the Core.

## Success Criteria

- Browser executable is detected.
- Browser can be launched by KayocktheOS.
- Browser opens to a local Bridge Dashboard.
- Documentation reflects the Core/Shell separation.
